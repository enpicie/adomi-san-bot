# Contributing

## Local Setup

Only Python 3.12 and git are assumed. There is no local Discord stack — local work is
unit tests; integration testing happens against the live dev deployment.

```bash
make test
```

Or manually:

```bash
python -m venv .venv
source .venv/bin/activate        # macOS/Linux
# .venv\Scripts\activate         # Windows

pip install setuptools
pip install -r requirements.txt
pip install -r requirements-dev.txt

export PYTHONPATH="$PYTHONPATH:$(pwd)/src"
# Windows: $env:PYTHONPATH="$env:PYTHONPATH;${PWD}\src"

pytest tests/
```

`make lint` runs [ruff](https://docs.astral.sh/ruff/) over `src jobs scripts tests`.

## Branch Conventions

- `feature/<short-description>` — new functionality
- `fix/<short-description>` — bug fixes
- `chore/<short-description>` — tooling, docs, refactors

## PR Flow

Pushing to **any non-main branch** triggers the Dev Pipeline
(`.github/workflows/development.yml`): `read-config` → `unit-test` (`make test`) →
`deploy` to the **dev** environment. So every push deploys your branch to dev — verify
behavior there with the dev bot before merging. Releases to prod happen by pushing a
`X.Y.Z` tag (`.github/workflows/release.yml`).

## How to Add a Command

1. Find (or create) the command group directory under `src/commands/<group>/`.
2. Add the handler function, then add an entry to the group's `mapping.py`
   (`CommandMapping` dict: name → function, description, params; add an
   `autocomplete_handler` key for autocomplete).
3. New group only: merge its mapping in `src/commands/command_map.py`.
4. Add unit tests under `tests/` mirroring the source path.
5. Register with Discord: run the **Register Discord Bot Slash Commands** workflow via
   `workflow_dispatch`, passing your command name (or `all`).

Official reference: https://discord.com/developers/docs/interactions/application-commands

## How to Add an AWS Service

Add the client to `src/aws_services.py` (`AWSServices`), which is constructed once at
Lambda startup and passed to every command handler. This keeps cold starts low and
lets tests inject mocks (moto) cleanly. Wire the resource itself in Terraform —
see "Adding new infrastructure" in the README.

## Common Mistakes

- **The 3-second rule.** Discord aborts interactions not answered within 3 seconds.
  Slow work (start.gg API fan-out, Sheets I/O) must be deferred or offloaded to SQS.
- **Jobs cannot import `src/`.** Each `jobs/<name>` zip is self-contained. Importing
  across the boundary passes locally and crashes in Lambda.
- **Keep MIRROR copies in sync.** Logic duplicated between `src/` and `jobs/` is
  marked with MIRROR comments — when you change one copy, change the other.
- **Don't inline Discord mention syntax or bare prints** — see `CLAUDE.md` Code Style
  (use `message_helper`, and `[name]`-prefixed prints in `src/` / `logger` in `jobs/`).
